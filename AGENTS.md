# 项目技术简介

使用 Astro@5 开发，页面用 tailwind 做样式实现，项目需要支持黑夜模式

## 项目架构

- src
  - assets # 静态资源文件
  - components # 全局通用组件
  - layouts # 全局布局
  - pages # 页面文件

## 开发规范/要点

默认布局取用 `src/layouts/Layout.astro`

文章页路径为 `/posts/[name]`

博客为 SSG 设计，不需要考虑 SSR 等情况。

pages 内导出的 `getStaticPaths` 函数所使用的变量，函数所在作用域必须都得在 `getStaticPaths` 内，不能引用外部的变量或者函数。

如果需要大规模移动文件时，可以直接在根目录编写一个脚本，再在对话里执行

### 路径规范

只有一个页面且确定不会有子路径时才可以直接创建路径对应的 astro 文件，否则必须要创建为 `xxx/index.astro` 形式

## 文章目录结构

文章存储于 `src/content/blog` 下，其中：

- 无图片或任何额外静态资源的文章直接用 `[标题].md` 存储
- 否则采用以下模式：

  `[标题]/index.md` 或 mdx 存储

  其次按照静态资源类型进行文件夹分类：
  - assets # 静态图像等资源
  - components # 页面组件
  - styles # 页面样式
